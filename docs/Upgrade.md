# 🚀 Agent Forge OS: Upgrade & Roadmap

본 문서는 Agent Forge OS의 핵심 가치와 향후 확장 가능한 4가지 주요 시나리오를 정의합니다.

## 1. 게임 개발 자동화 (인디 게임 기획 → 완성)
현재 프로젝트의 핵심 용도입니다. 사용자가 게임 아이디어만 입력하면 5명의 에이전트가 파이프라인으로 협력하여 플레이 가능한 게임을 생성합니다.

- **사용자 입력:** "탑다운 뷰 로그라이크, 마법사가 주인공, 3스테이지"
- **자동 파이프라인:**
  - **Planner:** GDD(Game Design Document) 생성 (스테이지 구성, 밸런스 수치, UX 흐름)
  - **Architect:** 기술 스펙 (Canvas 2D 렌더링, 충돌 시스템, 상태머신 설계)
  - **Compiler:** 태스크 분해 (파일 단위 구현 순서, 의존성 그래프)
  - **Worker:** HTML/CSS/JS 코드 작성, 에셋 생성
  - **Auditor:** 실행 테스트, 버그 리포트, 밸런스 피드백
- **가치:** 프로그래밍 경험이 없는 기획자도 아이디어만으로 프로토타입을 빠르게 검증 가능.

---

## 2. 소프트웨어 아키텍처 자동 설계
에이전트 역할을 범용 소프트웨어로 확장합니다. `employees.ts` 등의 역할 구조를 재매핑하여 활용합니다.

- **사용자 입력:** "실시간 채팅 앱, 1만 동시접속, React + Node.js"
- **에이전트 역할 재매핑:**
  - **Planner:** 요구사항 분석, 유저 스토리, MVP 범위 정의
  - **Architect:** 시스템 아키텍처 (WebSocket vs SSE, DB 선택, 인프라 구조도)
  - **Compiler:** API 명세서, 데이터 모델, 디렉토리 구조 생성
  - **Worker:** 보일러플레이트 코드, 핵심 모듈 구현
  - **Auditor:** 아키텍처 리뷰 (병목점, 보안 취약점, 확장성 평가)
- **가치:** 신규 프로젝트 시작 시 아키텍처 설계 및 초기 세팅 시간 대폭 단축.

---

## 3. 멀티-에이전트 협력 시스템 시뮬레이션
Agent Forge OS 자체를 에이전트 연구 및 교육 플랫폼으로 활용합니다.

- **연구 목적:** 서로 다른 LLM 조합의 협력 효율 비교 (예: GPT-4o vs Claude Sonnet vs Gemini Flash)
- **모니터링 요소:**
  - `AgentData`를 통한 실시간 상태 추적
  - 활동 로그의 시계열 관찰
  - 토큰 사용량 및 응답 시간 대시보드
- **가치:** 멀티 에이전트 시스템의 동작을 직관적으로 이해하고 최적의 협력 패턴을 실험하는 환경 제공.

---

## 4. 기술 문서 자동 생성
에이전트 파이프라인을 문서 생성 및 코드베이스 분석 특화로 전환합니다.

- **사용자 입력:** 기존 코드베이스 URL 또는 소스코드
- **실행 프로세스:**
  - **Planner:** 문서 범위 및 대상 독자 정의
  - **Architect:** 코드 분석 (모듈 구조, 의존성, 데이터 흐름)
  - **Compiler:** 문서 구조 설계 (목차, 섹션별 깊이)
  - **Worker:** 실제 문서 작성 (README, API reference, Architecture guide 등)
  - **Auditor:** 문서 품질 검증 및 가독성 평가
- **가치:** 지속적 문서화(Continuous Documentation) 구현을 통해 개발 효율 증대.

---

## 🎯 종합: 범용적 아키텍처의 강점
Agent Forge OS의 뼈대는 다음과 같은 이유로 강력한 범용성을 가집니다.

1. **역할 분리:** `employees.ts` 수정만으로 도메인 전환 가능
2. **콜백 패턴:** UI와 로직 분리로 다양한 출력 형태 지원
3. **상태 관리:** `WindowContext`를 통한 중앙 집중형 에이전트 관리
4. **시각적 모니터링:** 진행 상황을 직관적으로 시각화
5. **모듈식 윈도우:** 용도에 따라 UI 구성요소 유연하게 확장

**기획 → 설계 → 실행 → 검증**이라는 보편적 워크플로우를 에이전트 팀 구조와 실시간 UI로 풀어낸 것이 본 프로젝트의 핵심 경쟁력입니다.
