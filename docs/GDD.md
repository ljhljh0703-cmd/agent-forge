# Game Design Document — Slime Survivors

## 1. 개요
- **프로젝트명**: Slime Survivors
- **장르**: 뱀서라이크 (Vampire Survivors 스타일) 미니 버전
- **한 줄 요약**: 사방에서 몰려오는 슬라임을 피하며 자동 공격으로 생존하는 1분 서바이벌 게임

## 2. 아트 스타일
- CSS 기반 이모지+도형 렌더링 (이미지 에셋 없음)
- 다크 배경 + 네온 글로우
- 플레이어: 🗡️, 적: 🟢, 경험치: 💎

## 3. 핵심 메카닉
- **이동**: WASD 또는 방향키로 플레이어 이동, 맵 경계 제한
- **자동 공격**: 가장 가까운 적에게 0.8초마다 투사체 자동 발사
- **적 스폰**: 화면 밖 랜덤 위치에서 2초마다 슬라임 생성, 시간이 갈수록 빈도 증가
- **적 AI**: 플레이어를 향해 직선 이동 (속도 느림)
- **충돌**: 적과 닿으면 HP -1, HP 0이면 게임 오버
- **경험치**: 적 처치 시 💎 드롭, 수집 시 EXP 획득
- **레벨업**: EXP 바 가득 차면 공격 속도 +20% (최대 3회)
- **타이머**: 60초 생존하면 승리

## 4. 타겟 플랫폼
- 데스크탑 브라우저
- 모바일 터치 미지원 (데스크탑 키보드 전용)

## 5. 의사결정 로그
- Canvas 대신 DOM 렌더링 선택 → CSS 애니메이션 활용, 개발 속도 우선
- 프레임워크 없이 Vanilla JS → 의존성 제로, 즉시 실행 가능
- 60초 제한 → 짧은 세션으로 증명용에 적합
- 레벨업은 최대 3회로 제한 → 복잡도 관리
